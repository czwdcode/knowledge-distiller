## 使用方式

### 1. 配置文件设置 (`config.json`)
首先，新建一个 `config.json` 文件。这个文件告诉程序去哪里找视频、把文件存到哪，以及使用什么模型。以下是配置文件的完整结构和说明：
```json
{
    "urls": [
        "https://youtu.be/Fzk1Lwna",
        "https://youtu.be/Fzk1Lwna"
    ],
    "audio_path": "audio",
    "srt_path": "srt",
    "out_path": "output",
    "model_path": "本地faster-whisper模型路径",
    "cookiefile": "账号的cookies路径",
    "subtitle_langs": [
        "zh",
        "ai-zh"
    ]
}
```
**关键参数解读：**
- **urls**：在这里填入你想处理的视频链接，支持填入多个。
- **路径设置**：
    - `audio_path`：存放下载音频的文件夹（默认为 "audio"）。
    - `srt_path`：存放原始字幕文件的文件夹（默认为 "srt"）。
    - `out_path`：最终输出结果的文件夹（默认为 "output"）。
- **模型与认证**：
    - `model_path`：如果你使用本地的 faster-whisper 模型，这里需要填写模型的具体路径。
    - `cookiefile`：如果视频需要登录才能访问，这里要提供你账号 cookies 文件的路径。
- **subtitle_langs**：指定你要下载的字幕语言，这个是候选列表只要找到一个就会直接下载，并且只会下载最先找到的那个。

----

### 2. 核心脚本功能介绍

配置好文件后，你可以使用以下三个脚本来完成不同的任务：

#### 下载字幕：`subtitle_downloader.py`
这是主入口脚本。它的作用是读取上面创建的 `config.json` 配置文件，根据里面的链接和设置，自动去 YouTube 下载对应的字幕文件。

#### 格式清洗：`clean_srt_for_LLM.py`
下载下来的字幕通常是 `.srt` 格式，里面包含时间轴和序号，不适合直接喂给大语言模型（LLM）处理。这个脚本的作用就是把下载的 srt 文件进行格式化，剔除多余信息，只保留纯文字内容，方便后续分析或使用。

#### 语音转文字：`faster-whisper.py`
如果目标视频没有现成的字幕，或者你需要自己生成字幕，就需要用到这个脚本。它调用 faster-whisper 模型，将视频的音频转换为文字。使用前请确保在配置文件中正确设置了 `model_path`。
